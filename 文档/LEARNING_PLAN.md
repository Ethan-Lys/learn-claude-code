```
┌─────────────────────────────────────────────────────────────────┐
│                     工具系统架构                                 │
│                                                                 │
│   ┌─────────────┐                                               │
│   │ Tool 定义    │  = name + description + input_schema         │
```

```python
TOOLS = [
    {
        "name": "bash",
```

**理解三要素**：
- `name`: 工具名称，LLM 会用这个名字调用
- `description`: 描述，LLM 通过这个理解工具用途
- `input_schema`: 参数定义，告诉 LLM 需要传什么参数

#### Step 3: 理解工具处理器

找到 `TOOL_HANDLERS`：

```python
TOOL_HANDLERS = {
    "bash": handle_bash,
    "read_file": handle_read_file,
    "write_file": handle_write_file,
    "edit_file": handle_edit_file,
}
```

找到工具执行的核心逻辑：

```python
for block in response.content:
    if block.type == "tool_use":
        handler = TOOL_HANDLERS[block.name]  # 根据名字找处理器
        output = handler(block.input)        # 执行处理器
```

**关键理解**：这是一个**分发模式**（Dispatch Pattern）

```
┌─────────────────────────────────────────────────────────────────┐
│                     Todo 系统 vs 无规划 Agent                    │
│                                                                 │
│   无规划 Agent:                                                 │
│   用户: "重构这个文件，添加类型提示、docstring、main guard"      │
│   Agent: 嗯...先做类型提示...然后...等等，做了啥来着？           │
│   结果: 可能遗漏某些任务，或重复做同一件事                       │
│                                                                 │
│   有规划 Agent:                                                 │
│   用户: "重构这个文件..."                                       │

│          1. [pending] 添加类型提示                              │
│          2. [pending] 添加 docstring                            │
│          3. [pending] 添加 main guard                           │
│   Agent: 开始执行任务1...完成！标记 completed                   │
│   Agent: 开始执行任务2...完成！标记 completed                   │
│   Agent: 开始执行任务3...完成！标记 completed                   │
│   结果: 所有任务都完成，不会遗漏                                │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 核心理解

```
【Level 1 完成后的理解】

Agent = 模型 + Harness（代码框架）
- 模型：负责思考和决策（调用哪个工具、传什么参数、什么时候结束）
- Harness：负责执行（运行工具、管理消息历史、控制循环）

Agent Loop 是 Harness 的核心：
1. 调用 LLM，传入 messages 和 tools
2. 检查 stop_reason
3. 如果是 tool_use，执行工具，把结果追加到 messages，回到步骤 1
4. 如果不是 tool_use，任务完成

一句话总结：Agent 就是一个能自己决定"下一步做什么"的程序。
```
